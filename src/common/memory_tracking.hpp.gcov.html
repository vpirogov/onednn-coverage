      36              : /* Memory tracking capabilities
      37              :  *
      38              :  * The main purpose of this header file is to provide uniform way to register
      39              :  * required memory for a scratchpad at a primitive descriptor creation time
      40              :  * and then easily access it having only the base address of the scratchpad.
      41              :  *
      42              :  * Primitives might contain multiple disjoint parts that require temporary
      43              :  * buffers (known as scratchpad) during their execution. A primitive descriptor
      44              :  * should summarize all the needs into one single number -- the buffer size
      45              :  * that would be requested from a user. At execution time, the corresponding
      46              :  * primitive will receive a base pointer to a scratchpad. It then needs to
      47              :  * provide each part of algorithm the corresponding piece of memory. Three main
      48              :  * challenges here are:
      49              :  * 1. Track correct offset (from the base scratchpad address) for each piece
      50              :  * 2. Algorithm might require that different memory pieces to be aligned, so
      51              :  *    the scratchpad size is no more just a sum of size of the corresponding
      52              :  *    subparts.
      53              :  * 3. While a primitive is responsible for its scratchpad, the implementation
      54              :  *    might use some other basic blocks (e.g. cpu_reducer) that also require
      55              :  *    scratchpad memory. So there should be a simple way of passing the
      56              :  *    information back and forth between the main algorithm (a primitive) and
      57              :  *    auxiliary stuff that lives completely separately from it (e.g. reducer).
      58              :  *
      59              :  * To address these challenges this header file provides 3 abstractions:
      60              :  * 1. registry_t  -- the class to store the information about requested memory.
      61              :  *                   The information includes required size and desired
      62              :  *                   alignment for each piece. This class is also responsible
      63              :  *                   for computing the right offset to a given piece using the
      64              :  *                   base pointer.
      65              :  *                   This class is basically a ledger with all entries.
      66              :  *                   Lives in primitive descriptors.
      67              :  *
      68              :  * 2. registrar_t -- the interface to a registry_t to book memory. Used at
      69              :  *                   primitive descriptor creation time only. Contains a
      70              :  *                   reference to the corresponding *mutable* registry.
      71              :  *                   Always modifiable.
      72              :  *                   Allows chaining (using prefixes).
      73              :  *
      74              :  * 3. grantor_t   -- the interface to a registry_t to access memory. Used at
      75              :  *                   primitive execution time only. Contains a reference to
      76              :  *                   the corresponding *constant* registry and base pointer.
      77              :  *                   Always constant.
      78              :  *                   Allows chaining (using prefixes).
      79              :  *
      80              :  * Both registrar_t and grantor_t allow chaining with extra prefix provided.
      81              :  * The feature is useful when a primitive offload a part of computations to
      82              :  * some other primitives which require their own scratchpad space
      83              :  * (e.g. reducer). Prefixes are used to avoid key collision in cases when
      84              :  * multiple sub-primitive (e.g. multiple reducers) are used.
      85              :  *
      86              :  * A short example below demonstrates how to use aforementioned classes. In it
      87              :  * the main primitive is convolution that uses scratchpad for keeping padded
      88              :  * bias. It also needs a reducer, that needs its own space as well.
      89              :  *
