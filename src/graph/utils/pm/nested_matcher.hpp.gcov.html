      35              : //
      36              : // Nested pattern matcher is greedy graph pattern matcher that is based
      37              : // on limited look ahead.
      38              : // Matching start with a seed graph op and pattern graph.
      39              : // Pattern graph serves as a map for guiding the matching process and
      40              : // while matching, individual graph ops will be paired up with a pb_op
      41              : // in the pattern graph. pb_op_t encodes which edges need to be matched
      42              : // (topological constraints) and what other properties a a paired graph
      43              : // op should possess(attribute constraints).
      44              : // The pattern graph itself is a nested graph where nesting happens
      45              : // at alternation and repetition nodes.
      46              : //
      47              : // Each pattern graph level has a match_context associated with it.
      48              : // A match_context holds the pattern graph in current context and
      49              : // the in/out port map used to map upper-level contexts.
      50              : //
      51              : // A "binding" is pre step for graph op to pb_op_t pairing. At any pattern
      52              : // level, some pattern nodes are not pb_op. binding is between a
      53              : // graph op and a pattern node (not necessarily a pb_op).
      54              : // Matching is driven by binding. If binding is not to a pb_op,
      55              : // resolve_node() will be called which invokes matching at the nesting
      56              : // pattern level. This process will invoke match_graph, match_alternation
      57              : // and match_repetition. If binding is to a pb_op, match_node() is called,
      58              : // which checks graph op attributes with the pb_op's decision_functions
      59              : // and bind neighboring ops and corresponding pattern nodes.
      60              : //
