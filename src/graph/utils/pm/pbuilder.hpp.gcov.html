     243              : // "pb_graph_t" represents a group of pb_op_ts and also serves as a pb_node_t
     244              : // anywhere And provides a way to limit interface by limiting ports
     245              : // (placeholders) to outside of pb_graph_t.
     246              : // Nested/Hierarchical pb_nodes are useful for expressing patterns beyond fixed
     247              : // pb_graph_t. Regular expression like extension may works on a unit larger
     248              : // than a single pb_node_t.
     249              : // So a concept that represent grouping is going to be useful.
     250              : // pb_graph_t defines a way to forward input/output of the group
     251              : // to input/output of individual pb_nodes.
     252              : // For example, pb_graph_t "G" below wraps two connected pb_nodes "MUL" and
     253              : // "ADD" Collectively, G defines three inputs and one output. The three inputs
     254              : // of "G" are mapped to (pb_graph_t inner) inputs of "MUL" and "ADD"
     255              : // The single output of "G" maps to the single output of "ADD"
     256              : // Now, this "G" can used as part of a bigger pattern by connecting through
     257              : // the three inputs and one output just defined.
     258              : // Also, "G" can declare output ports which provides a way for backends to
     259              : // declare which outputs can be produced by compiled kernels for the pattern.
     260              : // Declaring output ports is important for exposing backend's ability to handle
     261              : // side outputs.
     262              : //    ----------------------
     263              : //    |   ------   -----   |
     264              : // 0- | 0-| MUL|---|ADD|   |
     265              : //    | 1-|    |  0|   |-0 |
     266              : // 1- |   ------   |   |   |-0
     267              : //    |          1-|   |   |
     268              : // 2- |            -----   |
     269              : //    ----------------------
     270              : //          pb_graph_t "G"
     271              : //
     272              : // G:IN0->MUL:IN0, G:IN1->MUL:IN1, G:IN2->ADD:IN1
